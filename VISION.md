# Loom — 产品愿景

## 一句话
macOS 原生的照片墙应用。用本地 AI 把用户的照片**编织**成一面有美感、有节奏、有情绪的墙。每次点击都是一次不重复的惊喜。

---

## 核心体验

### 零 → 有
打开时墙面是空的。用户点"组合"按钮，Loom 从照片库里自动挑一组、排一版，立刻呈现一整面墙。不需要任何预设、不需要拖拽。

### 每次都不一样
- 同样的照片库，每次点击产出的布局都不同
- 随机有约束：色彩平衡、构图比例、节奏感必须满足美学规则，然后在规则内做扰动
- "再来一次"（Shuffle）和"切换风格"是主交互，比任何编辑按钮都大

### 自动分组逻辑（用户可切换主轴）
- **颜色和谐**：同色系 / 互补色 / 渐变 / 统一色温
- **意境相近**：通过语义 embedding（CLIP-style）聚类出情绪/主题相近的一组
- **场景/人脸**：同一地点、同一批人
- **构图互补**：尺寸、留白、主次关系互相补位

### 风格预设
- **拼贴 Collage**：重叠、旋转、撕边感，手工质感
- **杂志 Editorial**：大图主导 + 小图点缀，黑白分明的版式感
- **画廊 Gallery**：克制的黄金比例网格，留白充足
- **织物 Tapestry**（默认）：justified 版式，每行统一高度，像织物的经纬
- **极简 Minimal**：每次只出 3–5 张，强对比
- **复古 Vintage**：宝丽来质感、白边、轻微倾斜

---

## 编辑自由度

自动生成是起点，不是终点。生成完用户可以：
- 拖动、缩放、旋转、替换单张
- 锁定某几张为"必须保留"，再 shuffle 其余
- 微调全局色调 / 间距 / 对齐
- 一键撤销到纯自动版本

### 收藏布局
- 喜欢的布局可以收藏，起名字，存缩略图
- 收藏的布局可以再应用到其他照片集
- 可以导出为图像 / PDF / 打印尺寸

---

## 技术栈

- **UI**：Swift + SwiftUI，精细交互处用 AppKit bridge
- **渲染**：重图片区域走 Metal / CALayer，保证大墙面的帧率
- **本地 AI**（全部离线）：
  - Vision 框架：`VNGenerateImageFeaturePrintRequest`（视觉相似）、场景分类、人脸
  - Core Image：主色提取、色温、对比度分析
  - Core ML（可选）：语义 embedding，做"意境"相似度
- **聚类**：HDBSCAN 或层次聚类，Swift 原生实现
- **数据源**：
  - PhotoKit 接系统相册
  - 用户自选文件夹，自建 SQLite 索引（路径、特征、缩略图缓存）
- **布局引擎**：自研，风格预设各自一套约束 + 一套扰动算法

---

## 产品原则

1. **原生优先**：性能和质感向 Apple 一方应用（Photos、Pages、Keynote）看齐，不用跨平台框架
2. **美学优先**：默认态必须好看。用户不需要调节就能得到顶级结果
3. **惊喜优先**：自动 > 手动。随机要有，但必须在美学约束内
4. **离线优先**：所有 AI / 算法跑在本地，不依赖网络，也不上传照片
5. **隐私优先**：沙盒安全，本地索引加密，用户可一键清空

- [ ] 主版式默认：Tapestry（justified）/ Masonry / 黄金比网格。当前倾向 Tapestry 作为默认，其他作为风格切换

- [ ] 数据源：PhotoKit 和用户文件夹并存，哪个是 v1 的主路径？倾向用户文件夹（自由度高、沙盒摩擦小）

## 路线图（草稿）

1. **M1 索引闭环**：读文件夹 → Vision 提特征 → SQLite 缓存 → 缩略图网格浏览
2. **M2 最小可玩**：单一风格（Tapestry）+ 单一聚类（颜色）+ Shuffle 按钮
3. **M3 多风格**：加入 Collage / Editorial / Gallery / Minimal / Vintage
4. **M4 意境聚类**：接入 Core ML embedding，"意境"成为独立主轴
5. **M5 编辑 + 收藏**：手动微调、锁定、收藏布局、命名、预览
6. **M6 导出 + 打印**：高分辨率导出、PDF、自定义物理尺寸
7. **M7 PhotoKit 源**：系统相册作为第二数据源

---

## 名字的由来

**Loom** = 织机。把一堆散落的照片当作经纬线，算法把它们编织成一面墙。一次编织不同于下一次，但每次都是一匹完整的织物。
